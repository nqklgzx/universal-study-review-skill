# 通用学科复习 Prompt Template Library

> 复制对应代码块，将【占位符】替换为具体学科、资料、章节或任务。

## S00｜设定复习助手角色

**用途：** 把 AI 固定为某门学科的复习教师或学习教练，使后续回答风格稳定，避免每次重新说明要求。

```text
你现在是我的【学科名称】复习助手，定位是该学科的资深教师/教练。你的任务是帮助我完成课程复习、资料整理、知识点理解、题型训练和考前速看。

请遵守以下要求：

一、角色定位
1. 以考试复习为目标，回答要准确、结构清晰、可用于复习。
2. 不只给结论，要说明结论依据、适用条件和常见误区。
3. 对题目解答优先采用标准考试书写流程。

二、资料使用规则
1. 如果我提供了教材、PPT、笔记、题库、录音总结或其他资料，回答前应优先基于这些资料。
2. 涉及页码、章节、图表、公式、题目条件时，必须先定位资料来源。
3. 若资料不足以支持结论，必须说明缺失信息，不能自行补充成确定事实。

三、解题输出格式
遇到题目时，优先按以下结构回答：
1. 考点定位
2. 解题思路
3. 具体解答
4. 最终答案
5. 易错点提示

四、概念讲解格式
遇到知识点问题时，优先按以下结构回答：
1. 准确结论
2. 概念本质
3. 公式/定义/适用条件
4. 典型题型
5. 易混淆点
6. 复习建议

五、输出风格
1. 使用中文。
2. 结构化输出。
3. 不省略关键推导。
4. 不使用空泛总结代替具体内容。
5. 不确定时先说明不确定原因。
```

## S01｜遍历资料并建立查阅清单

**用途：** 在正式复习前，把所有资料变成一个“索引地图”。后续 AI 回答问题时先查索引，再去对应文件查具体内容。

```text
请遍历我提供的所有【学科名称】资料，生成一份 Markdown 查阅清单，方便后续 AI 回答问题时先查阅该清单，再去对应文件中定位内容。

要求：

一、覆盖范围
1. 列出所有已提供文件。
2. 标明每个文件的类型，例如教材、PPT、笔记、题库、作业、录音总结、考试说明等。
3. 标明每个文件是否可提取文字、是否包含扫描图像、是否需要回原图核对。

二、文件结构
对每一个文件分别整理：
1. 文件名；
2. 文件类型；
3. 主要内容；
4. 章节结构；
5. 可查阅的知识点；
6. 适合用于哪些复习任务；
7. 如果有页码、幻灯片页码、章节号，请保留定位信息。

三、课程知识脉络
按课程章节或知识模块列出：
1. 模块名称；
2. 对应资料文件；
3. 对应章节或页码；
4. 主要知识点；
5. 常见题型；
6. 复习优先级。

四、使用规则
在文档开头写明后续 AI 使用本清单的规则：
1. 回答问题前先定位章节；
2. 再定位对应资料；
3. 涉及公式、图、表、题目条件时回原文件核对；
4. 不确定时说明缺失信息，不能自行编造。

五、输出要求
1. 输出为 Markdown；
2. 用清晰标题层级，方便目录跳转；
3. 不要只做概括，要尽量保留文件结构；
4. 不要把无法验证的内容写成确定结论。
```

## S02｜建立全课程知识脉络

**用途：** 把整门课从“文件堆”整理成“知识地图”，明确各章之间的关系、学习顺序和考试重点。

```text
请根据我提供的【学科名称】资料，整理整门课程的知识脉络。目标是让我快速理解这门课的整体结构，以及各章节之间的关系。

要求：

一、总体框架
1. 用一棵知识树展示整门课的结构。
2. 每个一级模块说明它解决什么问题。
3. 每个二级模块说明它包含哪些知识点。

二、章节关系
说明各章节之间的逻辑关系：
1. 哪些是基础概念；
2. 哪些是核心方法；
3. 哪些是应用；
4. 哪些是考试题型集中区域；
5. 哪些章节之间容易混淆或前后关联明显。

三、复习优先级
对每个模块标注复习优先级：
1. 高频核心；
2. 重要但偏理解；
3. 低频补充；
4. 仅需了解。

四、资料定位
每个模块后面写明：
1. 对应文件；
2. 对应章节；
3. 对应页码或幻灯片页码；
4. 需要回原图核对的内容。

五、输出要求
1. 输出 Markdown；
2. 标题层级清晰；
3. 不要写成泛泛总结；
4. 每个模块都要说明“学它是为了解决什么问题”。
```

## S03｜按章节或模块整理资料结构

**用途：** 对单章、单模块或单份 PPT 进行压缩整理，解决“PPT 页数多、例子多、公式散”的问题。

```text
请帮我整理【学科名称】中【章节/模块名称】的资料脉络。该部分资料内容较多，可能包含大量例题、图片、公式或重复说明。请把它整理成适合复习的 Markdown 文档。

要求：

一、章节定位
1. 本章属于整门课的哪个部分；
2. 本章主要解决什么问题；
3. 本章和前后章节有什么联系。

二、结构整理
按资料原有顺序整理：
1. 每一部分的标题；
2. 该部分讲了什么；
3. 该部分的核心概念；
4. 该部分的重要程度；
5. 对应页码、PPT页码或资料位置。

三、公式与结论
对每个知识点整理：
1. 核心公式；
2. 公式适用条件；
3. 二级结论；
4. 推导思路；
5. 常见变形；
6. 易错点。

四、题型对应
说明每个知识点常见于什么题型：
1. 概念判断题；
2. 计算题；
3. 证明题；
4. 综合题；
5. 图表分析题；
6. 材料题或论述题。

五、输出要求
1. 输出 Markdown；
2. 保留资料页码或位置；
3. 不要只概括，要按资料结构整理；
4. 对公式、图表、例题必须提示是否需要回原资料核对；
5. 最后给出本章复习路径。
```

## S04｜提取公式、定义、结论与方法

**用途：** 从章节资料中提取考试最常用的“硬信息”：公式、定义、判定条件、方法步骤。

```text
请从【章节/模块名称】中提取适合考试复习的公式、定义、关键结论和方法步骤。

要求：

一、定义
整理本章所有重要定义：
1. 定义名称；
2. 标准表述；
3. 适用范围；
4. 判断关键词；
5. 易混淆对象。

二、公式
整理本章核心公式：
1. 公式名称；
2. 公式表达式；
3. 每个符号含义；
4. 适用条件；
5. 常见变形；
6. 使用时的限制；
7. 典型题型。

三、方法步骤
整理本章常用解题方法：
1. 方法名称；
2. 适用题型；
3. 标准步骤；
4. 关键判断点；
5. 常见错误；
6. 检查方法。

四、辨析
对容易混淆的概念或公式做对比：
1. 相同点；
2. 不同点；
3. 题目中如何判断；
4. 考试中如何书写。

五、输出要求
1. 输出 Markdown；
2. 按“定义—公式—方法—辨析—易错点”组织；
3. 公式必须写清适用条件；
4. 不要只列公式，要说明什么时候用。
```

## S05｜融合课堂重点、录音、教师要求

**用途：** 把老师上课强调、录音整理、考试说明、划重点内容整合进复习资料，形成真正贴合考试的优先级。

```text
请根据我提供的课堂录音总结、课堂笔记或教师要求，结合【学科名称】资料，整理一份“课堂重点与资料定位”文档。

要求：

一、信息分类
把老师相关信息分为：
1. 明确强调的重点；
2. 明确说过可弱化或不重点关注的内容；
3. 老师提到但未明确说明重要程度的内容；
4. 与考试范围、题型、答题方式有关的要求。

二、资料定位
对每一条课堂信息，定位到：
1. 对应章节；
2. 对应教材/PPT/讲义；
3. 对应页码或幻灯片页码；
4. 对应知识点；
5. 是否需要回原图、公式或例题核对。

三、复习优先级
根据老师表述标注：
1. 重点掌握；
2. 熟练会做；
3. 理解即可；
4. 了解即可；
5. 暂不优先。

四、约束
1. 只有老师明确说过的内容，才标为“重点”或“可弱化”。
2. 老师没提到的内容按正常复习处理，不能写成不用复习。
3. 不确定的地方必须标注“不确定原因”。
4. 不能把工具初步总结中的模糊表述当成确定结论。

五、输出要求
1. 输出 Markdown；
2. 每条重点都要有资料定位；
3. 最后给出基于教师要求的复习顺序。
```

## S06｜建立题型库与标准解题模板

**用途：** 把知识点转化为考试题型，形成每类题的固定解法、书写格式和检查方法。

```text
请根据【学科名称】的资料和题目，建立一份题型库与标准解题模板。

要求：

一、题型分类
按考试常见形式整理题型：
1. 题型名称；
2. 对应章节；
3. 核心考点；
4. 常见问法；
5. 常见陷阱。

二、标准解题模板
对每一类题写出：
1. 读题时先看什么；
2. 如何判断考点；
3. 应使用哪些定义、公式或定理；
4. 标准解题步骤；
5. 何时需要分情况讨论；
6. 最终答案应包含哪些信息；
7. 如何检查答案。

三、典型例题定位
每类题至少列出：
1. 对应资料来源；
2. 例题编号或页码；
3. 该题体现的核心方法；
4. 是否适合作为模板题。

四、易错点
每类题整理：
1. 容易漏掉的条件；
2. 容易用错的公式；
3. 容易写错的符号；
4. 容易混淆的概念；
5. 检查方法。

五、输出要求
1. 输出 Markdown；
2. 每类题都要有“标准解题步骤”；
3. 不要只列题目，要抽象出方法；
4. 语言要适合考试复习。
```

## S07｜深入讲解单个知识点

**用途：** 当某个概念看不懂时，用统一结构让 AI 深入解释，而不是只给简短定义。

```text
我现在看不懂【知识点名称】。请按照考试复习的方式讲清楚。

要求：

一、先给结论
用简洁准确的话说明这个知识点到底是什么。

二、解释本质
说明：
1. 它为什么被提出；
2. 它解决什么问题；
3. 它和前后知识点的关系；
4. 它在题目中通常起什么作用。

三、公式与条件
如果涉及公式，请说明：
1. 公式是什么；
2. 每个符号代表什么；
3. 公式从哪里来；
4. 适用条件是什么；
5. 不满足条件时会出现什么问题。

四、对比辨析
如果它容易和其他概念混淆，请对比：
1. 相同点；
2. 不同点；
3. 题目中如何区分；
4. 哪种问法对应哪种方法。

五、考试应用
说明：
1. 常见题型；
2. 解题时第一步看什么；
3. 标准书写方式；
4. 常见错误；
5. 检查方法。

六、输出要求
1. 不要只给定义；
2. 不要跳过关键推理；
3. 尽量结合一个简单例子说明；
4. 如果资料条件不足，请指出需要补充什么。
```

## S08｜标准化解答具体题目

**用途：** 对具体题目进行考试标准化解答，训练答题过程和书写格式。

```text
请按【学科名称】考试标准流程解答这道题。

题目如下：
【粘贴题目或上传图片】

要求：

一、考点定位
1. 判断本题属于哪一章或哪个模块；
2. 判断本题考查哪些定义、公式、方法或定理；
3. 说明它属于哪类题型。

二、读题分析
1. 提取已知条件；
2. 明确所求内容；
3. 标出隐含条件；
4. 如果题目信息不足，先说明缺失条件。

三、解题思路
1. 说明为什么选择该方法；
2. 如果有多种方法，说明最适合考试书写的方法；
3. 说明需要注意的条件或分情况讨论。

四、具体解答
1. 写出使用的公式、定义或定理；
2. 按步骤推导；
3. 关键变形不能跳步；
4. 如果涉及图、表、区间、边界、条件、单位或符号，必须写清楚。

五、最终答案
1. 清楚给出最终结果；
2. 如果答案有适用范围、条件、单位、区间或其他限制，必须一并写出。

六、易错点提示
1. 本题最容易错在哪里；
2. 如何检查答案；
3. 哪些条件不能漏。

输出要求：
1. 使用结构化标题；
2. 不要只给结果；
3. 不要省略关键推导；
4. 语言要适合考试复习。
```

## S09｜整理错题、易错点和辨析点

**用途：** 把做错或看不懂的题沉淀为错题库，防止同类错误重复出现。

```text
请帮我整理这道错题，目标是让我以后遇到同类题不会再错。

题目：
【粘贴题目】

我的错误解法或疑问：
【粘贴你的过程或疑问】

标准答案或参考答案：
【如果有就粘贴，没有就写“暂无”】

要求：

一、错因定位
1. 我错在概念理解、公式选择、条件判断、计算过程、符号书写还是审题；
2. 具体是哪一步错；
3. 为什么这一步容易错。

二、正确解法
1. 给出标准解题流程；
2. 写清关键公式和适用条件；
3. 说明正确答案如何得到。

三、同类题识别
1. 以后看到什么关键词或结构，就应该想到这种方法；
2. 哪些题看起来相似但不能用这种方法；
3. 如何快速判断。

四、易错点卡片
请最后输出一张“错题卡片”，包括：
1. 错题标签；
2. 核心考点；
3. 错因一句话；
4. 正确方法一句话；
5. 检查方法；
6. 需要背下来的提醒。

输出要求：
1. 不要只改正答案；
2. 重点解释为什么会错；
3. 最后要形成可复习的错题卡片。
```

## S10｜生成考前速看资料

**用途：** 把长期整理的章节脉络、公式、题型、重点、易错点压缩成考前快速翻阅资料。

```text
请根据我提供的【学科名称】复习资料，整理一份考前速看资料。

输入资料包括：
1. 章节脉络；
2. 公式与定义；
3. 题型库；
4. 老师重点；
5. 错题和易错点；
6. 考试范围。

要求：

一、总体要求
1. 内容面向考前快速翻阅；
2. 不写空泛总结；
3. 只保留高价值内容；
4. 每条内容尽量短，但不能丢失关键条件；
5. 适合打印、背诵或临考前快速查看。

二、结构
按章节或模块整理，每章包括：
1. 本章高频考点；
2. 必背定义；
3. 核心公式；
4. 公式适用条件；
5. 常见题型；
6. 标准方法；
7. 易错点；
8. 检查方法。

三、重点处理
1. 老师明确强调的内容优先放前面；
2. 高频题型优先放前面；
3. 易错点单独集中整理；
4. 相似概念要做简短辨析。

四、输出格式
1. 输出 Markdown；
2. 标题层级清晰，方便目录跳转；
3. 公式、定义、题型、易错点分块呈现；
4. 最后给出考前最后一遍复习路线。
```

## S11｜后续问答时按资料定位回答

**用途：** 让 AI 在每次回答具体问题前，先根据已有查阅清单、章节脉络和资料定位，再进行讲解或解题。

```text
请回答下面这个【学科名称】问题。回答前请先根据已有资料查阅清单和章节脉络定位知识点，再进行解释或解题。

问题：
【粘贴问题】

要求：

一、先定位
1. 判断问题属于哪个章节或模块；
2. 说明应查阅哪些资料；
3. 如果涉及页码、图表、公式或题目条件，请先说明需要核对的位置。

二、再回答
根据问题类型选择输出方式：

如果是概念题：
1. 准确结论；
2. 概念本质；
3. 公式或定义；
4. 适用条件；
5. 易混淆点；
6. 典型题型。

如果是计算题或证明题：
1. 考点定位；
2. 解题思路；
3. 具体解答；
4. 最终答案；
5. 易错点提示。

如果是资料定位题：
1. 对应文件；
2. 对应章节；
3. 对应页码或位置；
4. 该位置包含的内容；
5. 后续应如何使用。

三、约束
1. 不要脱离资料随意发挥；
2. 资料不足时说明不足；
3. 不确定的地方不能写成确定结论。
```

## S12｜迭代更新资料库和提示词

**用途：** 随着复习推进，把新资料、新题型、新错题、新老师重点持续合并进复习系统。

```text
请根据新资料和新要求，迭代更新我已有的【学科名称】复习文档。

已有文档：
【粘贴或上传旧文档】

新增资料/新要求：
【粘贴或上传新内容】

要求：

一、更新原则
1. 保留旧文档中仍然正确且有价值的内容；
2. 合并新增资料；
3. 修正错误或不准确表述；
4. 删除重复、空泛或低价值内容；
5. 不要因为新增内容而破坏原有结构。

二、版本对比
请说明：
1. 新增了什么；
2. 修改了什么；
3. 删除或弱化了什么；
4. 哪些地方仍然不确定；
5. 哪些内容需要回原资料核对。

三、输出要求
1. 输出更新后的完整 Markdown；
2. 保持标题层级清晰；
3. 保持可通过目录跳转；
4. 关键提示词保持可复制；
5. 不要只输出修改建议，要输出可直接使用的新版本。
```
