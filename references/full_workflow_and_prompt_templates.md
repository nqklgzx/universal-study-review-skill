# 通用学科复习标准流程与提示词模板

> 使用目的：把任意学科的复习资料、课堂重点、题目训练和考前速看整理成一套可复制、可迭代、可追溯的 AI 辅助复习工作流。  
> 适用范围：任何课程、考试、证书、竞赛、专业课、通识课、开卷考试、闭卷考试、技能型学习。  
> 使用方式：在 Markdown 软件中通过目录跳转到对应流程，复制该步骤下的提示词，把方括号中的内容替换为你的具体学科、资料、章节或任务。

---

## 0. 总体使用原则

### 0.1 核心目标

这套流程的目标不是让 AI 简单“总结资料”，而是建立一个完整的复习系统：

1. 先建立资料索引；
2. 再梳理知识结构；
3. 再结合课堂/老师/考试要求筛重点；
4. 再训练题型与标准答题流程；
5. 最后形成考前速看材料；
6. 后续提问时先定位资料，再解答问题。

### 0.2 工作流总览

```text
S00 设定复习助手角色
→ S01 遍历资料并建立查阅清单
→ S02 建立全课程知识脉络
→ S03 按章节/模块整理资料结构
→ S04 提取公式、定义、结论与方法
→ S05 融合课堂重点、录音、教师要求
→ S06 建立题型库与标准解题模板
→ S07 深入讲解单个知识点
→ S08 标准化解答具体题目
→ S09 整理错题、易错点和辨析点
→ S10 生成考前速看资料
→ S11 后续问答时按资料定位回答
→ S12 迭代更新资料库和提示词
```

### 0.3 全局回答要求

后续所有提示词都应默认遵守以下原则：

```text
1. 不编造资料中没有的信息。
2. 不把不确定内容写成确定结论。
3. 涉及原文、页码、图表、公式、题目条件时，必须先定位资料来源。
4. 如果资料无法支持结论，应明确说明缺失信息。
5. 解题时必须写清方法选择、关键公式、推导过程和最终答案。
6. 概念题要讲清本质、适用条件、典型题型和易错点。
7. 输出要结构化，便于复习、复制、归档。
```

# S00｜设定复习助手角色

## 用途

把 AI 固定为某门学科的复习教师或学习教练，使后续回答风格稳定，避免每次重新说明要求。

## 适用场景

- 新建一个学科复习项目；
- 准备长期使用 AI 辅助复习；
- 需要统一回答格式、解题格式和资料查阅规则。

## 输入

- 学科名称；
- 考试类型；
- 是否开卷；
- 资料类型；
- 回答风格要求。

## 输出

- 一份可长期使用的角色提示词；
- 一套统一的答题格式；
- 一套资料查阅规则。

## 复制即用提示词

```text
你现在是我的【学科名称】复习助手，定位是该学科的资深教师/教练。你的任务是帮助我完成课程复习、资料整理、知识点理解、题型训练和考前速看。

请遵守以下要求：

一、角色定位
1. 以考试复习为目标，回答要准确、结构清晰、可用于复习。
2. 不只给结论，要说明结论依据、适用条件和常见误区。
3. 对题目解答优先采用标准考试书写流程。

二、资料使用规则
1. 如果我提供了教材、PPT、笔记、题库、录音总结或其他资料，回答前应优先基于这些资料。
2. 涉及页码、章节、图表、公式、题目条件时，必须先定位资料来源。
3. 若资料不足以支持结论，必须说明缺失信息，不能自行补充成确定事实。

三、解题输出格式
遇到题目时，优先按以下结构回答：
1. 考点定位
2. 解题思路
3. 具体解答
4. 最终答案
5. 易错点提示

四、概念讲解格式
遇到知识点问题时，优先按以下结构回答：
1. 准确结论
2. 概念本质
3. 公式/定义/适用条件
4. 典型题型
5. 易混淆点
6. 复习建议

五、输出风格
1. 使用中文。
2. 结构化输出。
3. 不省略关键推导。
4. 不使用空泛总结代替具体内容。
5. 不确定时先说明不确定原因。
```

## 质量检查

- 是否明确了学科和考试目标；
- 是否规定了资料优先级；
- 是否规定了解题结构；
- 是否规定了不确定信息的处理方式。

# S01｜遍历资料并建立查阅清单

## 用途

在正式复习前，把所有资料变成一个“索引地图”。后续 AI 回答问题时先查索引，再去对应文件查具体内容。

## 适用场景

- 有大量 PPT、教材、PDF、笔记、题库；
- 不清楚每个文件包含什么；
- 后续希望 AI 能先定位资料再回答。

## 输入

- 教材；
- PPT；
- 笔记；
- 课后习题；
- 题库；
- 录音整理；
- 老师划重点文件；
- 历年题或模拟题。

## 输出

建议输出 Markdown 文件，内容包括：

```text
1. 资料覆盖范围
2. 文件总表
3. 每个文件的结构
4. 每个章节对应文件
5. 每类题型对应资料
6. 无法提取文本的文件说明
7. 后续查阅规则
```

## 复制即用提示词

```text
请遍历我提供的所有【学科名称】资料，生成一份 Markdown 查阅清单，方便后续 AI 回答问题时先查阅该清单，再去对应文件中定位内容。

要求：

一、覆盖范围
1. 列出所有已提供文件。
2. 标明每个文件的类型，例如教材、PPT、笔记、题库、作业、录音总结、考试说明等。
3. 标明每个文件是否可提取文字、是否包含扫描图像、是否需要回原图核对。

二、文件结构
对每一个文件分别整理：
1. 文件名；
2. 文件类型；
3. 主要内容；
4. 章节结构；
5. 可查阅的知识点；
6. 适合用于哪些复习任务；
7. 如果有页码、幻灯片页码、章节号，请保留定位信息。

三、课程知识脉络
按课程章节或知识模块列出：
1. 模块名称；
2. 对应资料文件；
3. 对应章节或页码；
4. 主要知识点；
5. 常见题型；
6. 复习优先级。

四、使用规则
在文档开头写明后续 AI 使用本清单的规则：
1. 回答问题前先定位章节；
2. 再定位对应资料；
3. 涉及公式、图、表、题目条件时回原文件核对；
4. 不确定时说明缺失信息，不能自行编造。

五、输出要求
1. 输出为 Markdown；
2. 用清晰标题层级，方便目录跳转；
3. 不要只做概括，要尽量保留文件结构；
4. 不要把无法验证的内容写成确定结论。
```

## 质量检查

- 是否每个文件都被列出；
- 是否每个文件都有结构说明；
- 是否标注了资料定位路径；
- 是否区分了“可提取文本”和“需要看图核对”；
- 是否建立了后续回答规则。

# S02｜建立全课程知识脉络

## 用途

把整门课从“文件堆”整理成“知识地图”，明确各章之间的关系、学习顺序和考试重点。

## 适用场景

- 刚开始复习；
- 想知道整门课在讲什么；
- 想先建立宏观框架；
- 需要把知识点串起来。

## 输入

- 查阅清单；
- 教材目录；
- PPT 目录；
- 课程大纲；
- 考试范围。

## 输出

```text
1. 全课程知识树
2. 各章节核心任务
3. 章节之间的逻辑关系
4. 高频考点
5. 复习顺序建议
6. 各章节资料定位
```

## 复制即用提示词

```text
请根据我提供的【学科名称】资料，整理整门课程的知识脉络。目标是让我快速理解这门课的整体结构，以及各章节之间的关系。

要求：

一、总体框架
1. 用一棵知识树展示整门课的结构。
2. 每个一级模块说明它解决什么问题。
3. 每个二级模块说明它包含哪些知识点。

二、章节关系
说明各章节之间的逻辑关系：
1. 哪些是基础概念；
2. 哪些是核心方法；
3. 哪些是应用；
4. 哪些是考试题型集中区域；
5. 哪些章节之间容易混淆或前后关联明显。

三、复习优先级
对每个模块标注复习优先级：
1. 高频核心；
2. 重要但偏理解；
3. 低频补充；
4. 仅需了解。

四、资料定位
每个模块后面写明：
1. 对应文件；
2. 对应章节；
3. 对应页码或幻灯片页码；
4. 需要回原图核对的内容。

五、输出要求
1. 输出 Markdown；
2. 标题层级清晰；
3. 不要写成泛泛总结；
4. 每个模块都要说明“学它是为了解决什么问题”。
```

## 质量检查

- 是否讲清楚整门课主线；
- 是否讲清楚章节关系；
- 是否能通过目录快速跳转；
- 是否每个模块都有资料定位；
- 是否区分了复习优先级。

# S03｜按章节或模块整理资料结构

## 用途

对单章、单模块或单份 PPT 进行压缩整理，解决“PPT 页数多、例子多、公式散”的问题。

## 适用场景

- 一章 PPT 很长；
- 想知道每部分讲什么；
- 想把一章变成可复习的结构化笔记；
- 想保留页码定位。

## 输入

- 某一章 PPT；
- 某一章教材；
- 某一章讲义；
- 查阅清单；
- 课堂重点。

## 输出

```text
1. 本章目标
2. 本章知识脉络
3. 每部分内容说明
4. 重点程度
5. 公式与结论
6. 页码定位
7. 题型对应
8. 易错点
```

## 复制即用提示词

```text
请帮我整理【学科名称】中【章节/模块名称】的资料脉络。该部分资料内容较多，可能包含大量例题、图片、公式或重复说明。请把它整理成适合复习的 Markdown 文档。

要求：

一、章节定位
1. 本章属于整门课的哪个部分；
2. 本章主要解决什么问题；
3. 本章和前后章节有什么联系。

二、结构整理
按资料原有顺序整理：
1. 每一部分的标题；
2. 该部分讲了什么；
3. 该部分的核心概念；
4. 该部分的重要程度；
5. 对应页码、PPT页码或资料位置。

三、公式与结论
对每个知识点整理：
1. 核心公式；
2. 公式适用条件；
3. 二级结论；
4. 推导思路；
5. 常见变形；
6. 易错点。

四、题型对应
说明每个知识点常见于什么题型：
1. 概念判断题；
2. 计算题；
3. 证明题；
4. 综合题；
5. 图表分析题；
6. 材料题或论述题。

五、输出要求
1. 输出 Markdown；
2. 保留资料页码或位置；
3. 不要只概括，要按资料结构整理；
4. 对公式、图表、例题必须提示是否需要回原资料核对；
5. 最后给出本章复习路径。
```

## 质量检查

- 是否保持原资料结构；
- 是否写明页码或位置；
- 是否标注重要程度；
- 是否整理公式和适用条件；
- 是否能用于考前复习。

# S04｜提取公式、定义、结论与方法

## 用途

从章节资料中提取考试最常用的“硬信息”：公式、定义、判定条件、方法步骤。

## 适用场景

- 需要公式表；
- 需要概念表；
- 需要方法清单；
- 需要考前背诵材料；
- 需要区分相似公式。

## 输入

- 某章资料；
- 某章脉络整理；
- 教材/PPT；
- 老师重点。

## 输出

```text
1. 定义清单
2. 公式清单
3. 公式适用条件
4. 判定条件
5. 方法步骤
6. 相似概念辨析
7. 易错点
```

## 复制即用提示词

```text
请从【章节/模块名称】中提取适合考试复习的公式、定义、关键结论和方法步骤。

要求：

一、定义
整理本章所有重要定义：
1. 定义名称；
2. 标准表述；
3. 适用范围；
4. 判断关键词；
5. 易混淆对象。

二、公式
整理本章核心公式：
1. 公式名称；
2. 公式表达式；
3. 每个符号含义；
4. 适用条件；
5. 常见变形；
6. 使用时的限制；
7. 典型题型。

三、方法步骤
整理本章常用解题方法：
1. 方法名称；
2. 适用题型；
3. 标准步骤；
4. 关键判断点；
5. 常见错误；
6. 检查方法。

四、辨析
对容易混淆的概念或公式做对比：
1. 相同点；
2. 不同点；
3. 题目中如何判断；
4. 考试中如何书写。

五、输出要求
1. 输出 Markdown；
2. 按“定义—公式—方法—辨析—易错点”组织；
3. 公式必须写清适用条件；
4. 不要只列公式，要说明什么时候用。
```

## 质量检查

- 公式是否有适用条件；
- 定义是否有判断关键词；
- 方法是否能直接用于做题；
- 是否区分了相似概念；
- 是否避免只列清单不解释。

# S05｜融合课堂重点、录音、教师要求

## 用途

把老师上课强调、录音整理、考试说明、划重点内容整合进复习资料，形成真正贴合考试的优先级。

## 适用场景

- 有录音文字稿；
- 有课堂笔记；
- 老师说过“这里重点”“这里不考”“这里只需了解”；
- 想把老师要求映射到 PPT 或教材位置。

## 输入

- 录音总结；
- 课堂笔记；
- 老师划重点；
- PPT；
- 教材；
- 已整理章节脉络。

## 输出

```text
1. 老师强调点
2. 老师弱化点
3. 对应资料页码
4. 考试优先级
5. 复习建议
6. 不确定内容说明
```

## 复制即用提示词

```text
请根据我提供的课堂录音总结、课堂笔记或教师要求，结合【学科名称】资料，整理一份“课堂重点与资料定位”文档。

要求：

一、信息分类
把老师相关信息分为：
1. 明确强调的重点；
2. 明确说过可弱化或不重点关注的内容；
3. 老师提到但未明确说明重要程度的内容；
4. 与考试范围、题型、答题方式有关的要求。

二、资料定位
对每一条课堂信息，定位到：
1. 对应章节；
2. 对应教材/PPT/讲义；
3. 对应页码或幻灯片页码；
4. 对应知识点；
5. 是否需要回原图、公式或例题核对。

三、复习优先级
根据老师表述标注：
1. 重点掌握；
2. 熟练会做；
3. 理解即可；
4. 了解即可；
5. 暂不优先。

四、约束
1. 只有老师明确说过的内容，才标为“重点”或“可弱化”。
2. 老师没提到的内容按正常复习处理，不能写成不用复习。
3. 不确定的地方必须标注“不确定原因”。
4. 不能把工具初步总结中的模糊表述当成确定结论。

五、输出要求
1. 输出 Markdown；
2. 每条重点都要有资料定位；
3. 最后给出基于教师要求的复习顺序。
```

## 质量检查

- 是否区分了老师明确说过和推测内容；
- 是否每条重点都有资料定位；
- 是否没有把未提及内容误写成不复习；
- 是否保留了不确定信息。

# S06｜建立题型库与标准解题模板

## 用途

把知识点转化为考试题型，形成每类题的固定解法、书写格式和检查方法。

## 适用场景

- 进入刷题阶段；
- 需要总结常考题型；
- 想把题目训练标准化；
- 想形成答题模板。

## 输入

- 课后题；
- 题库；
- 历年题；
- 章节例题；
- 课堂例题；
- 章节知识点清单。

## 输出

```text
1. 题型分类
2. 每类题对应考点
3. 标准解题步骤
4. 常用公式
5. 典型例题
6. 易错点
7. 检查方法
```

## 复制即用提示词

```text
请根据【学科名称】的资料和题目，建立一份题型库与标准解题模板。

要求：

一、题型分类
按考试常见形式整理题型：
1. 题型名称；
2. 对应章节；
3. 核心考点；
4. 常见问法；
5. 常见陷阱。

二、标准解题模板
对每一类题写出：
1. 读题时先看什么；
2. 如何判断考点；
3. 应使用哪些定义、公式或定理；
4. 标准解题步骤；
5. 何时需要分情况讨论；
6. 最终答案应包含哪些信息；
7. 如何检查答案。

三、典型例题定位
每类题至少列出：
1. 对应资料来源；
2. 例题编号或页码；
3. 该题体现的核心方法；
4. 是否适合作为模板题。

四、易错点
每类题整理：
1. 容易漏掉的条件；
2. 容易用错的公式；
3. 容易写错的符号；
4. 容易混淆的概念；
5. 检查方法。

五、输出要求
1. 输出 Markdown；
2. 每类题都要有“标准解题步骤”；
3. 不要只列题目，要抽象出方法；
4. 语言要适合考试复习。
```

## 质量检查

- 是否覆盖主要题型；
- 是否每类题都有标准步骤；
- 是否能直接指导做题；
- 是否有检查方法；
- 是否有例题定位。

# S07｜深入讲解单个知识点

## 用途

当某个概念看不懂时，用统一结构让 AI 深入解释，而不是只给简短定义。

## 适用场景

- 概念不理解；
- 公式不知道为什么成立；
- 不知道什么时候用；
- 和相似概念混淆；
- 题目中不会判断。

## 输入

- 知识点名称；
- 所在章节；
- 你不理解的具体句子或公式；
- PPT截图或教材截图；
- 已有疑问。

## 输出

```text
1. 准确结论
2. 本质解释
3. 公式来源
4. 适用条件
5. 与相似概念对比
6. 题目判断方法
7. 易错点
```

## 复制即用提示词

```text
我现在看不懂【知识点名称】。请按照考试复习的方式讲清楚。

要求：

一、先给结论
用简洁准确的话说明这个知识点到底是什么。

二、解释本质
说明：
1. 它为什么被提出；
2. 它解决什么问题；
3. 它和前后知识点的关系；
4. 它在题目中通常起什么作用。

三、公式与条件
如果涉及公式，请说明：
1. 公式是什么；
2. 每个符号代表什么；
3. 公式从哪里来；
4. 适用条件是什么；
5. 不满足条件时会出现什么问题。

四、对比辨析
如果它容易和其他概念混淆，请对比：
1. 相同点；
2. 不同点；
3. 题目中如何区分；
4. 哪种问法对应哪种方法。

五、考试应用
说明：
1. 常见题型；
2. 解题时第一步看什么；
3. 标准书写方式；
4. 常见错误；
5. 检查方法。

六、输出要求
1. 不要只给定义；
2. 不要跳过关键推理；
3. 尽量结合一个简单例子说明；
4. 如果资料条件不足，请指出需要补充什么。
```

## 质量检查

- 是否先给了准确结论；
- 是否解释了为什么；
- 是否讲清适用条件；
- 是否能帮助做题；
- 是否有辨析和易错点。

## 输出

```text

2. 正确方法
3. 触发条件
4. 同类题判断

提取已知条件、所求内容、隐含条件和限制条件。

三、解题思路
说明为什么使用该方法，是否需要分情况讨论。

四、具体解答
写出公式、定义或定理，并逐步推导，不跳步。

五、最终答案
清楚给出结果，包含必要条件、范围、单位或限制。

六、易错点提示
说明本题容易错在哪里，以及如何检查。
```
