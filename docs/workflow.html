Concept

提示词和 Skill 的关系

这套项目的主要内容仍然是提示词。你可以直接复制首页的 S00–S12 提示词完成任何具体任务。

Skill 工作流相当于“自动执行层”：当你希望 Codex 在一个课程仓库中持续读取资料、生成文档、更新索引、维护错题和考前速看时，就调用 Skill。

直接用提示词

适合单次整理、单题讲解、单章总结、临时问答。复制对应提示词即可。

调用 Skill

适合长期项目、资料很多、需要 Codex 自动写入 Markdown 文件、希望形成可维护复习仓库。

When

什么时候调用 Skill

资料很多

你有教材、PPT、讲义、题库、录音总结，需要先生成资料查阅清单。

想做成长期项目

你希望每次新增资料后都能更新索引、章节脉络、错题和速看资料。

想让 Codex 写文件

你希望直接在仓库中生成 Markdown 文件，而不是只在聊天窗口里查看答案。

想统一输出规范

你希望所有题解、概念讲解、章节总结都按照同一套考试复习格式输出。

Workflow

Skill 会按什么流程执行

1先检查仓库和资料结构
2确认学科、考试目标和资料优先级
3建立资料查阅清单
4整理章节脉络和公式方法
5融合老师重点和考试要求
6建立题型模板和错题规则
7生成考前速看资料
8后续新增资料时迭代更新

Install

安装方式

方式一：skills 安装器

npx skills@latest add nqklgzx/universal-study-review-skill

方式二：Windows PowerShell 远程安装到当前仓库

powershell -NoProfile -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/nqklgzx/universal-study-review-skill/main/scripts/bootstrap-codex-skill.ps1 | iex"

Invocation

如何调用

首次配置课程仓库

Use /setup-universal-study-review to configure this course repository.

正式执行复习工作流

使用 $universal-study-review，把这门课的资料整理成复习查阅清单、章节脉络、题型模板和考前速看资料。
Use $universal-study-review to build a material index and review workflow for this course.

Scenarios

不同场景怎么说

想整理一堆资料

使用 $universal-study-review，先遍历当前仓库中的课程资料，生成资料查阅清单。要求保留文件名、章节结构、页码或资料位置、可查阅知识点和后续回答规则。

想按章节生成复习笔记

使用 $universal-study-review，基于已有资料查阅清单，按章节生成复习脉络。每章包含核心概念、公式条件、典型题型、易错点和资料定位。

想生成题型库

使用 $universal-study-review，根据课程资料和题目，建立题型库与标准解题模板。每类题写清识别方法、标准步骤、最终答案要求和检查方法。

想做考前速看

使用 $universal-study-review，根据章节脉络、公式清单、老师重点、题型库和错题，生成考前速看资料。内容要短，但不能丢失关键条件。

新增资料后更新

使用 $universal-study-review，读取新增资料并迭代更新已有复习文档。说明新增了什么、修改了什么、哪些内容仍需回原资料核对。

Rules

调用 Skill 时的执行规则

  • 先定位资料来源，再总结或解题。
  • 不编造页码、章节、公式、老师要求或题目条件。
  • 资料不足时明确说明缺失信息。
  • 长文档优先输出 Markdown，并保持标题层级清晰。
  • 题目解答必须包含方法选择、公式条件、推导步骤、最终答案和易错点。

提示词负责具体功能；Skill 负责把这些功能串成可执行、可维护的复习工作流。
